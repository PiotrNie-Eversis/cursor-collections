# Start here

**Cursor Collections** is a **Cursor-native** product engineering framework for the full SDLC:

> **Ideate → Implement → Review**

Each phase produces documented artifacts; the next step waits on **your approval** where the prompts require it.

:::tip New to this framework?
Follow the checklist below in order. Each step links to the detailed playbook — this page is the map, not the encyclopedia.
:::

## Setup checklist

1. **[Prerequisites](./prerequisites)** — [Cursor](https://cursor.com/), **Node.js ≥ 18** (MCP build), Git.
2. **[Installation](./installation)** — clone this repo or bootstrap a consumer project with `setup-cursor-local.sh`.
3. **[MCP setup](./mcp-setup)** — build `eversis-collections`, enable servers from `.cursor/mcp.json`.
4. **First prompt** — attach `@eversis-implement` or use `/eversis-implement` with your ticket or task text. See [Prompts overview](../prompts/overview).
5. **[Workflow overview](../workflow/overview)** — pick a variant (standard, UI, workshop, E2E, BA docs).
6. **Go deeper** — [Framework reference](../framework), [Agents overview](../agents/overview), [Skills overview](../skills/overview).
