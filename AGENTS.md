# Agent instructions (Cursor)

This repository implements the **Eversis Cursor** workflow: **Ideate → Implement → Review**, using Cursor rules, attachable markdown prompts, MCP, and procedural **`SKILL.md` packages** under **`.cursor/skills/`** (consumed in Agent through the **`eversis-collections` MCP** server — `eversis_*` skill tools plus `.docx` chapter tools `generate_summary_map`, `read_chapter`, `update_chapter`, `upload_to_sharepoint` — build `mcp/eversis-collections-mcp/` first).

- **Framework doc:** [documentation/cursor-collection.md](documentation/cursor-collection.md)
- **Project rules:** [.cursor/rules/](.cursor/rules/) — start with `eversis-agent-core.mdc` and edit `eversis-project-stack.mdc` per repo. For Implement / Review, optionally attach `eversis-engineering-manager.mdc` and `eversis-code-reviewer.mdc` when using the matching prompts.
- **Prompts:** Canonical **`eversis-*.md`** files live under **`.cursor/prompts/public/`** (user-facing) and **`.cursor/prompts/internal/`** (delegated). In Chat or Agent, **attach with `@` and the file stem** (e.g. **`@eversis-implement`**, **`@eversis-review`**) so Cursor can resolve the file by name. **Use** a repo path (e.g. **`.cursor/prompts/public/eversis-implement.md`**) only if the picker does not disambiguate. Optional: editor **snippets** in [`.vscode/eversis-prompts.code-snippets`](.vscode/eversis-prompts.code-snippets) expand short prefixes (e.g. `evimpl`) to the same **`@`** text. Catalog: [website/docs/prompts/overview.md](website/docs/prompts/overview.md).

**Prefixes:** The **`eversis-`** name is used for rules, prompt files, and **skill package directories** in this framework (e.g. `.cursor/skills/eversis-creating-skills/`). **Do not** register **`.cursor/skills/`** as Cursor **Agent Skills**; use the **`eversis-collections`** entry in [`.cursor/mcp.json`](.cursor/mcp.json) after building the local MCP package.

**Fine → QA comment (Implement contract):** When **`eversis-implement`** finishes agent-side work, declare **Fine** and output the QA draft **in the same response**, following [`.cursor/skills/eversis-qa-comment/SKILL.md`](.cursor/skills/eversis-qa-comment/SKILL.md) — see [workflow overview — Status: Fine](website/docs/workflow/overview.md). Normative procedure is **`SKILL.md`** or MCP `eversis_skills_get`; do not rely on [`website/docs/skills/qa-comment.md`](website/docs/skills/qa-comment.md) alone for output shape. Reinforced in **`.cursor/commands/eversis-implement.md`** and **`eversis-project-stack.mdc`** (always-on in this repo; downstream repos should mirror if they consume the framework).

**Business Manager Docs (Word):** Prompts **`eversis-ba-docs-planner`** / **`eversis-ba-docs-writer`** use the **`eversis-collections`** MCP for `.docx` chapter tools (`generate_summary_map`, `read_chapter`, `update_chapter`, `upload_to_sharepoint`). There is no separate `eversis-docs` MCP entry in `.cursor/mcp.json` — build **`mcp/eversis-collections-mcp/`** once for skills and Word tools together.
